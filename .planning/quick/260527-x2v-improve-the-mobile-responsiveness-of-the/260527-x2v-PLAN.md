<objective>
Improve mobile responsiveness of the Brokr dashboard. The header logo currently looks tiny on mobile (60px default → 36px @768px → 30px @420px) and the header layout at 420px stacks `.header-left` and `.header-right` onto two full-width rows, which wastes vertical space and looks unbalanced. Polish the Indexa tab KPI grid and charts for narrow viewports.

Purpose: Make the dashboard pleasant to use on iPhone (390px), Pixel/Android (412px), and tablet (768px) screens after the recent flex-wrap header fix made the logo look out of proportion.

Output: Updated `app/static/style.css` (primary) and possibly minor `app/static/index.html` markup tweak. No JS changes expected.
</objective>

<task type="auto">
  <name>Task 1: Fix header logo sizing and single-row header layout at mobile breakpoints</name>
  <files>app/static/style.css</files>
  <action>
Update the `@media (max-width: 768px)` and `@media (max-width: 420px)` blocks in `app/static/style.css` so that:

(a) The Brokr logo is visually proportionate at each breakpoint. Recommended target heights (executor may tune by ±4px after visual check): 60px desktop (unchanged) → 44-48px at 768px → 38-42px at 420px. Do NOT drop below 36px — that was the source of the "looks tiny" complaint. Verify with the human-verify checkpoint.

(b) At the 420px breakpoint, stop forcing `.header-left` and `.header-right` to `flex: 1 1 100%`. That rule makes them claim full width each, stacking into two rows when both could fit on one. Replace with `flex: 1 1 auto` (or `min-width: 0` on `.header-left` + `flex-shrink: 0` on `.header-right`) so the logo + tab-nav sit on one line and the action buttons sit beside them. The header may still grow taller due to icon-button padding, but it should not split into two visual rows of full-width blocks.

(c) At the 420px (and below) breakpoint, ensure action buttons in `.header-right` are icon-only (the `.btn-label` `display:none` rule from the 768px block already covers this — verify it cascades; if not, restate inside the 420px block). The buttons should be compact icon squares so DeGiro+Indexa tabs and 3-4 action icons all fit on one row at 390-412px width.

(d) Add a small breakpoint adjustment for very narrow viewports if needed: if after (a)-(c) the elements still overflow at 390px, reduce `.tab-btn` padding to `3px 8px` and `font-size` to `0.7rem` inside the 420px block. Avoid horizontal scrolling on the body (`overflow-x: hidden` already on body — keep it).

(e) Keep the existing `flex-wrap: wrap` on `.header` as a safety net so extreme widths (<360px) degrade gracefully, but tune the children so wrapping does NOT trigger at the target widths (390px and 412px).

DO NOT introduce a new structural HTML change unless step (b)+(c) cannot achieve a single-row layout via CSS alone. If HTML adjustment is required (e.g., adding a `header-actions-compact` wrapper), document why in the commit message.

Per the existing media-query structure, also bump `.indexa-charts` chart-wrap height to be slightly shorter at 420px (e.g., `height: 220px`) for better mobile fit — only if currently 260px feels too tall on a phone (executor discretion, verify visually).
