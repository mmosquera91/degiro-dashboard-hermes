**app/market_data.py** (line 1605), as inner closures. There is NO module-level `fetch_price_batch` and

- **Step 2 / "Stage 3"** (lines 1695-1742) — price batch. `unique_yf_symbols` is split into
  `eu_symbols = [s for s in unique_yf_symbols if "." in s]` and `us_symbols = [... if "." not in s]`
  (yf.download drops plain US tickers when suffixed EU tickers are in the same list, hence the split).
  Inner `_fetch_and_unpack(symbols)` (line 1709) chunks by `_CHUNK_SIZE = 25` and per chunk calls
  `yf.download(chunk, period="2d", auto_adjust=True, progress=False, threads=True)`, then builds
  `{yf_symbol: latest_close}` via `close_df = b["Close"]` and `sym in close_df.columns` (DataFrame) /
  `sym == close_df.name` (Series), guarded `except (ValueError, TypeError, KeyError): pass`, with a
  `price > 0` filter. Called once per group: `price_batch = {**_fetch_and_unpack(eu_symbols), **_fetch_and_unpack(us_symbols)}`.
- **Step 3 / "Stage 4"** (lines 1744-1786) — `enrich_position(pos, price_batch)` per position via
  `run_in_executor`. Inside `enrich_position` (line 1148, esp. 1470-1500), price is read from
  `price_batch[yf_symbol]` first; GBp pence→pounds (÷100) conversion happens THERE, gated on
  `ticker.info currency == "GBp"`. Price must keep flowing through `price_batch` keyed by yf_symbol,
  un-converted.
- **Post-batch pass / "Stage 6"** (lines 1788-1873) — `_needs_enrichment` selects positions where
  `_resolved_yf_symbol` set AND `price_source == "batch"` AND `rsi is None` AND `perf_30d is None`.
  Inner `_post_enrich_one(idx, pos)` (line 1807) does a PER-SYMBOL
  `ticker.history(period="1y", auto_adjust=True)` (via run_in_executor, batched in groups of 20, with a
  `period="3mo"` fallback when `len(hist) < 14`), then `close = hist["Close"]`; if `len(close) < 2`
  return pos; `pos["rsi"] = compute_rsi(close)`, `perf = _compute_performance(close)` (30d/90d/ytd/1y),
  and — only when `current_price = pos.get("current_price")` is not None and > 0 —
  `52w_low = round(hist["Low"].min(),4)`, `52w_high = round(hist["High"].max(),4)`,
  `distance_from_52w_high_pct = round(((current_price-hi)/hi)*100, 2)` when hi>0. Whole body try/excepted.

The per-symbol 1y loop (post-batch pass) is the dominant cost. T5 replaces BOTH the Step-2 2d fetch and
the per-symbol post-batch fetch with ONE batched `yf.download(symbols, period="1y", threads=True)` per
group, building a `{yf_symbol: 1y DataFrame}` map once, then deriving price locally (latest close →
`price_batch`) for Step 3 AND rsi/perf/52w for the post-batch pass from the same frames. Net: 2 batch
calls (EU + US) instead of 2 batch calls + ~N per-symbol calls.

Purpose: cut steady-state enrichment from ~9s to ~2-4s by eliminating the ~N per-symbol 1y round-trips.

Output: a single batched-1y download per group in `enrich_positions`, a small pure `derive_from_history`
helper (unit-testable), and tests covering the DataFrame-shape edge cases.

NOTE: T1-T4 are already implemented and committed (do NOT redo them). Step 2 already has threads=True +
chunking (T2). T5 changes WHAT is fetched (1y instead of 2d) and reuses the result for the post-batch
metrics instead of re-fetching per symbol.
