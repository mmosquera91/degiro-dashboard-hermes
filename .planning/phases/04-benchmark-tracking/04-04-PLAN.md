  <name>Task 1: Fix /api/hermes-context to work without portfolio session</name>
  <files>app/main.py</files>
  <action>
Modify the `/api/hermes-context` endpoint (lines 476-488) to remove the hard requirement on portfolio session presence.

Current code raises HTTPException 404 when portfolio is None:
```python
if portfolio is None:
    raise HTTPException(status_code=404, detail="No portfolio data loaded. Refresh your portfolio first.")
```

Change: Remove the 404 guard. Instead, pass an empty dict `{}` to `build_hermes_context()` when portfolio is None. The `build_hermes_context()` function in `context_builder.py` already handles empty/missing portfolio gracefully — it loads snapshots independently, computes benchmark data from them, and only sets attribution to empty list when positions are unavailable.

New endpoint logic:
```python
with _session_lock:
    portfolio = _session.get("portfolio")

# build_hermes_context handles empty portfolio — snapshots are sufficient
return build_hermes_context(portfolio if portfolio is not None else {})
```

The key insight: `build_hermes_context()` calls `load_snapshots()` which works without any portfolio session. Benchmark and attribution data can be computed from snapshots alone. Only the portfolio_summary, positions, top_candidates, and health_alerts sections require live portfolio data — those will be empty/minimal but the benchmark/attribution sections will still be populated.
</action>

```javascript
// After Chart initialization, check if benchmark series is empty
const benchmarkSeries = data?.benchmark_series || [];
if (benchmarkSeries.length === 0 && snapshots.length > 0) {

  const chartWrap = $(".benchmark-chart-wrap");

    chartWrap.innerHTML = '<div class="benchmark-empty">No benchmark data available. yfinance may be unavailable or benchmark symbol is invalid.</div>';
```
