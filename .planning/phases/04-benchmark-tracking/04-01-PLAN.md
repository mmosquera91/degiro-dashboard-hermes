# Attribution formula (D-11)

absolute_contribution = position_return * weight
direction = 1 if position_return >= 0 else -1
weight = position weight as decimal (e.g., 0.20 for 20%)

Create app/snapshots.py with these functions:

1. `save_snapshot(date_str: str, total_value_eur: float, benchmark_value: float, benchmark_return_pct: float)`:
   - Uses SNAPSHOT_DIR env var (default: "/data/snapshots")
   - Creates directory with parents=True, exist_ok=True
   - Writes {date_str}.json with fields: date, total_value_eur, benchmark_value, benchmark_return_pct
   - benchmark_value is indexed to 100 at the FIRST snapshot date (portfolio start), not the current date

2. `load_snapshots() -> list[dict]`:
   - Reads all {date}.json files from SNAPSHOT_DIR
   - Returns sorted by date ascending
   - Returns empty list if directory does not exist

3. `fetch_benchmark_series(start_date: str, end_date: str) -> list[dict]`:
   - Uses BENCHMARK_TICKER env var (default: "^GSPC")
   - Uses _yf_throttle() from market_data to respect rate limits
   - Uses yf.download(ticker, start=start_date, end=end_date, progress=False)
   - Returns list of {"date": "YYYY-MM-DD", "value": float} where value is indexed to 100 at start_date
   - start_date and end_date are in YYYY-MM-DD format

4. `compute_attribution(positions: list[dict], benchmark_return: float) -> list[dict]`:
   - For each position, compute:
     - position_return = perf_ytd or 0 if None
     - weight = position["weight"] / 100.0 (weight is stored as percentage)
     - direction = 1 if position_return >= 0 else -1
     - relative_contribution = round((position_return - benchmark_return) * weight * direction, 4)
     - absolute_contribution = round(position_return * weight, 4)
   - Returns list sorted by absolute_contribution descending (largest first)
   - Each dict has: name, symbol, relative_contribution, absolute_contribution
   - Handle None perf_ytd as 0

The snapshot's benchmark_value is indexed to 100 at the FIRST snapshot date (portfolio start). On each snapshot:
- If this is the first snapshot ever: benchmark_value = 100.0, benchmark_return_pct = 0.0
- If snapshots exist: fetch benchmark from earliest snapshot date to today, compute benchmark_value = (current_benchmark_price / price_at_first_snapshot) * 100

3. Add new endpoint /api/benchmark:
   ```python
   @app.get("/api/benchmark", dependencies=[Depends(verify_brok_token)])
   async def get_benchmark():
       """Return benchmark comparison data: snapshots, indexed series, and attribution."""
       # Load all snapshots
       snapshots = load_snapshots()
       if not snapshots:
           return {"snapshots": [], "benchmark_series": [], "attribution": [], "message": "No snapshots yet"}
       
       # Get date range from snapshots
       first_date = snapshots[0]["date"]
       today = datetime.now().strftime("%Y-%m-%d")
       
       # Fetch benchmark series
       benchmark_series = fetch_benchmark_series(first_date, today)
       
       # Get current portfolio for attribution
       with _session_lock:
           portfolio = _session["portfolio"]
       if portfolio is None:
           return {"snapshots": snapshots, "benchmark_series": benchmark_series, "attribution": [], "message": "No portfolio loaded"}
       
       # Compute attribution using current benchmark return (from last snapshot)
       latest_benchmark_return = snapshots[-1].get("benchmark_return_pct", 0) if snapshots else 0
       attribution = compute_attribution(portfolio.get("positions", []), latest_benchmark_return)
       
       return {
           "snapshots": snapshots,
           "benchmark_series": benchmark_series,
           "attribution": attribution,
       }
   ```

   The /api/benchmark response must include:
   - snapshots: array of snapshot objects (each has date, total_value_eur, benchmark_value, benchmark_return_pct)
   - benchmark_series: array of {"date": "YYYY-MM-DD", "value": float} indexed to 100 at first snapshot date
   - attribution: array sorted by absolute_contribution descending with fields name, symbol, relative_contribution, absolute_contribution
