In the get_portfolio() function (lines 316-360), restructure the read-check-return sequence so that _is_session_valid() is called INSIDE the with _session_lock block.

Current buggy pattern (lines 323-334):
```python
with _session_lock:
    portfolio = _session["portfolio"]
    if portfolio is not None:
        return portfolio

# BUG: _is_session_valid() reads _session WITHOUT the lock
if not _is_session_valid():
    raise HTTPException(...)
trading_api = _session["trading_api"]
```

Fixed pattern:
```python
with _session_lock:
    portfolio = _session["portfolio"]
    if portfolio is not None:
        return portfolio

    # _is_session_valid() now reads _session while lock is HELD
    if not _is_session_valid():
        raise HTTPException(status_code=401, detail="Session expired or not authenticated. Please reconnect via the UI.")
    trading_api = _session["trading_api"]
```

Key changes:
1. Move the `_is_session_valid()` call AND the `HTTPException` raise inside the `with _session_lock:` block
2. Keep `trading_api = _session["trading_api"]` inside the lock too
3. The exception should be raised inside the lock so the lock is released properly via the `with` statement
