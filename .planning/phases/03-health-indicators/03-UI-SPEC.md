## Color

**60/30/10 Design Rule:** The color palette follows a 60/30/10 distribution:
- 60% dominant: `#0f0f0f` (page background)
- 30% secondary: `#1a1a1a` (cards, surfaces)
- 10% accent: `#01696f` (primary actions, active states, ETF highlighting)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0f0f0f | Page background |
| Secondary (30%) | #1a1a1a | Cards, surfaces |
| Accent (10%) | #01696f | Primary actions, active states, ETF highlighting |
| Amber | #d97706 | Stock allocation, warning severity indicators |
| Green | #22c55e | Positive values, success states |
| Red | #ef4444 | Negative values, critical severity, destructive |

**Accent reserved for:**
- Primary button (`btn-primary`)
- Active filter/tab states
- Radar item backgrounds
- ETF chart segments
- Focus ring on form inputs
- Link color in session instructions

**Health alert severity color mapping (new):**
| Severity | Color | Background | Border |
|----------|-------|------------|--------|
| warn | #d97706 (amber) | rgba(217,119,6,0.12) | rgba(217,119,6,0.3) |
| critical | #ef4444 (red) | rgba(239,68,68,0.12) | rgba(239,68,68,0.3) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Connect" (already exists, no new CTA in this phase) |
| Empty state heading | "All systems healthy" |
| Empty state body | "No health alerts detected. Your portfolio looks balanced." |
| Error state | "Failed to load health alerts: [error message]. Try refreshing your portfolio." |
| Destructive confirmation | None in this phase — health alerts are read-only signals |

**Health alert card copy pattern:**
Each alert displays:
- Alert type label (e.g., "Concentration Risk", "Sector Weighting", "Drawdown", "Rebalancing")
- Severity badge: "Warning" or "Critical"
- Message: human-readable description
- Current value vs threshold (e.g., "AAPL at 23% — threshold 20%")

**Specific alert messages (from D-01 through D-18):**
| Type | Warn message template | Critical threshold |
|------|----------------------|--------------------|
| Concentration Risk | "{position} is {value}% of portfolio (threshold {threshold}%)" | >20% |
| Sector Weighting | "Sector {sector} is {value}% of portfolio (threshold {threshold}%)" | >40% |
| Drawdown | "Portfolio YTD return is {value}% (threshold {threshold}%)" | <-10% |
| Rebalancing | "ETF allocation at {value}% — target {target}% (±{threshold}pp)" | drift >5pp |

## Health Alerts UI Section (Phase 3 Scope)

### Placement
- Rendered in `renderDashboard()` in `app/static/app.js`
- Inserted between summary cards and charts section
- Consistent with existing section layout patterns

### Focal Point
The section title "Health Alerts" is the primary visual anchor — it appears first and uses `.section-title` typography (16px, weight 600), establishing the section's presence before the alert cards themselves.

### Visual Design
```
.health-alerts-section
  .section-title "Health Alerts"
  .health-alerts-list
    .alert-card.(warn|critical)
      .alert-header
        .alert-type-label
        .alert-severity-badge
      .alert-message
      .alert-detail (current value vs threshold)
```

### Component States
| State | Appearance |
|-------|------------|
| warn | amber left border (3px), amber icon, amber text |
| critical | red left border (3px), red icon, red text |
| no-alerts | muted check icon, "All systems healthy" copy |
| loading | hidden (health alerts computed inline with portfolio) |
| error | error message inline with retry approach |

### Alert Card Styling
```css
.alert-card {
  background: var(--surface);
  border: 1px solid var(--border);
  border-left: 3px solid [severity-color];
  border-radius: var(--radius-sm);
  padding: 12px 14px;
  margin-bottom: 8px;
}
.alert-card.warn { border-left-color: #d97706; }
.alert-card.critical { border-left-color: #ef4444; }
```

### Integration with Existing Components
- Reuse `.card` base styles where applicable
- Reuse `.section-title` typography for section header
- Reuse `.wl-item` pattern for alert detail line (label + value layout)
- No new spacing tokens — use existing 8px/12px/16px spacing from style.css
