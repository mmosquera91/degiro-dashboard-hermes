## Key Middleware Flow
1. check_session_cookie middleware (http middleware) checks for brokr_session cookie on all non-exempt paths
2. If cookie missing/invalid -> 303 redirect to /login, invalid cookie deleted
3. If cookie valid -> request passes through to route
4. /api/* routes have Depends(verify_brok_token) which checks Bearer token

## Key Test Patterns (from existing tests)
- conftest.py provides `client` fixture with noop lifespan
- conftest.py provides `with_auth_env` monkeypatch for env vars
- TestClient.post/get with cookies={} and headers={} for auth
- response.status_code, response.headers["location"], response.cookies

## Login Flow
POST /login (form data: password) -> 303 redirect to / on success, sets brokr_session cookie
GET /api/session-token -> returns {"token": BROKR_AUTH_TOKEN} if valid session cookie present
GET /api/portfolio -> returns 200 + portfolio data if both session cookie AND Bearer token valid
